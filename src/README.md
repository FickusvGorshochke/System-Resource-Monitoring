# sysmond — сервис системного профилирования для ЗОСРВ «Нейтрино»

**Тема ВКР:** Сервисы системного профилирования для операционных систем реального времени, обеспечивающие мониторинг временных характеристик и ресурсов исполнения задач

**Автор:** К.И. Ахметгареев, гр. 2381, кафедра МО ЭВМ, СПбГЭТУ «ЛЭТИ»

---

## Краткое описание

`sysmond` — резидентный сервис-демон для ЗОСРВ «Нейтрино» (КПДА, редакция 2021), регистрирующий устройство `/dev/sysmon` через паттерн Resource Manager. Сервис периодически опрашивает `/proc` и накапливает в кольцевом буфере историю состояний потоков, использования CPU и памяти. Клиентская утилита `sysmon_cli` обеспечивает доступ к собранным данным через стандартные POSIX-вызовы.

Назначение — заполнить нишу между лёгкими штатными утилитами (`pidin`, `hogs`), которые показывают только мгновенный срез, и тяжеловесными профилировщиками (`System Profiler`), требующими подключения к хост-ПК.

## Состав

| Компонент | Назначение |
|-----------|------------|
| `daemon/main.c` | Точка входа sysmond, разбор аргументов, daemon-режим |
| `daemon/collector.{c,h}` | Поток сбора метрик через `procfs` |
| `daemon/ringbuf.{c,h}` | Потокобезопасный кольцевой буфер фиксированного размера |
| `daemon/resmgr_handler.{c,h}` | Resource Manager на `/dev/sysmon`, обработчики `devctl` |
| `cli/sysmon_cli.c` | Клиентская утилита (snapshot, history, stats, config, set, top, cpuns) |
| `common/sysmon_protocol.h` | Общий протокол сервиса и клиентов |
| `Makefile` | Сборка через `qcc -V` для x86, ARMv7, ARMv8 |

## Сборка

Требуется установленный КПДА «Нейтрино» 2021. Перед сборкой выставить окружение:

```bash
export QNX_HOST=/opt/kpda2021a/host/linux64/x86_64
export QNX_TARGET=/opt/kpda2021a/target/neutrino
export PATH=$QNX_HOST/usr/bin:$PATH
```

Сборка под x86:

```bash
make ARCH=x86 DEBUG=1
```

Получаются бинарники `sysmond` и `sysmon_cli`.

## Развёртывание на целевой системе

```bash
make deploy QNX_HOST_ALIAS=qnx
```

Эквивалент:

```bash
scp sysmond sysmon_cli qnx:/usr/bin/
ssh qnx 'chmod +x /usr/bin/sysmond /usr/bin/sysmon_cli'
```

## Использование

### Запуск сервиса

```bash
# В фоне (по умолчанию)
sysmond -p 1000

# На переднем плане для отладки
sysmond -f -p 1000
```

Аргументы:
- `-p <мс>` — период опроса (10..60000, по умолч. 1000)
- `-H <ч>` — глубина истории (1..168, по умолч. 24)
- `-t <N>` — макс. потоков на снимок (1..256, по умолч. 8)
- `-P <prio>` — приоритет коллектора (по умолч. 10)
- `-f` — foreground

В режиме демона лог пишется в `/var/log/sysmond.log`.

### Клиентская утилита

```bash
sysmon_cli snapshot                # текущее состояние всех потоков
sysmon_cli stats                    # статистика буфера и сервиса
sysmon_cli config                   # текущая конфигурация
sysmon_cli set -p 500               # сменить период на 500 мс
sysmon_cli history -p PID -n 1000   # история по процессу
sysmon_cli top                      # непрерывное обновление
sysmon_cli cpuns PID                # сырое CPU-время процесса в нс
```

## Архитектура

```
┌──────────────────┐  poll каждые N мс  ┌──────────────┐
│  Collector       │ ─────────────────▶ │              │
│  (SCHED_RR)      │   sysmon_record_t  │  Ring buffer │
└──────────────────┘                    │   (fixed)    │
        │                               └──────┬───────┘
        │ procfs                               │
        ▼                                      │
┌──────────────────┐                           │
│  /proc/<pid>/as  │                           │
│  DCMD_PROC_*     │                           │
└──────────────────┘                           │
                                               │ snapshot/history
                                               ▼
                                        ┌──────────────┐
                                        │  Resource    │
                                        │  Manager     │ ◀── /dev/sysmon
                                        │  (devctl)    │
                                        └──────────────┘
                                               ▲
                                               │ POSIX
                                               │
                                        ┌──────────────┐
                                        │  sysmon_cli  │
                                        └──────────────┘
```

Сбор данных через `DCMD_PROC_TIDSTATUS` / `DCMD_PROC_INFO` / `DCMD_PROC_MAPDEBUG_BASE` / `DCMD_PROC_MAPINFO`. Состояние и накопленное CPU-время потоков читается из ядра атомарно (`procfs_status.sutime`).

## Характеристики (измерены)

- **Overhead на CPU**: 0.03% при 1 Гц, 0.33% при 10 Гц (на VM x86, 107 потоков)
- **Память**: 37 МБ ring-буфер при настройках по умолчанию
- **Глубина истории**: 24 часа при 1 Гц
- **Покрытие**: до 128 потоков на снимок, неограниченно в истории

## Платформы

- **Сборка:** Linux (Ubuntu 24.04, КПДА Devkit 2021 academic)
- **Цель:** ЗОСРВ «Нейтрино» 2021, x86 (32-bit), ARMv7
